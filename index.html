DaySync

AI 기반 스마트 일정 관리 앱 | 캡스톤 프로젝트 | 2025.03 ~ 11
개요
일정 관리의 번거로움을 해결하고자 시작한 프로젝트로, AI 챗봇을 통한 자연스러운 대화로 일정을 관리하고 실시간 교통정보와 날씨를 제공하는 라이프스타일 앱입니다. 사용자가 복잡한 UI 없이 대화만으로 일정을 등록하고 최적 경로를 안내받을 수 있도록 설계했습니다.
기여한 부분
  • FastAPI로 RESTful API 서버 개발 및 AWS EC2 배포
  • 일정 CRUD, AI 챗봇 연동, 날씨/교통정보 조회 등 REST API 엔드포인트 설계 및 구현
  • 공공데이터포털 API 장애 시 기상청 API로 자동 전환하여 서비스 안정성 확보
  • Gemini AI API 재시도 로직 구현으로 일시적 오류 해결
  • 네이버 지도 API 연동으로 실시간 버스 경로 안내 기능 구현
트러블 슈팅

문제: 국토교통부 TAGO 버스 API 통합 시, 동일한 개념(정류장 순서)을 나타내는 필드가 API 응답마다 달라 일관된 처리 불가능

구체적 상황:

• 정류장 순서: `ord`, `seq`, `sequence`, `nodeord`, `stationSeq` 등 5개 이상의 필드명 존재
• 데이터 타입: `nodeord`는 String, 나머지는 int로 반환
• 방향 정보: `updown`, `directCd`, `routeDirection` 등 여러 필드 혼재

해결: `TagoBusRouteStationResponse` 클래스에 모든 가능한 필드를 선언하고, `getValidOrder()` 우선순위 메서드로 유효한 값 추출

배운 점: 외부 API 통합 시 응답 필드의 일관성을 보장할 수 없으므로, 방어적 코딩과 우선순위 기반 폴백 로직의 중요성 체득

Tech Stack: Python (FastAPI), Android (Java), AWS EC2, Gemini AI API, Naver Maps API, 공공데이터포털

SOS (Style of Simple)

패션 초보 남성을 위한 온라인 쇼핑몰 | 2024.03~06, 09~11
개요
패션에 관심 없었던 남성들이 옷을 고를 때 겪는 어려움을 해결하고자 시작한 프로젝트입니다. 20~30대 남성을 타겟으로 키워드 기반 추천 시스템을 통해 체형·상황·스타일에 맞는 제품을 쉽게 찾을 수 있도록 개발한 Spring Boot 기반 전자상거래 플랫폼입니다.
기여한 부분
  • Spring Boot MVC 패턴 기반 계층형 아키텍처 설계
  • 회원/판매자 권한별 접근 제어, 장바구니, 주문 처리 시스템 개발
  • 키워드 기반 상품 추천 알고리즘으로 체형·상황·분위기별 맞춤 제품 검색 구현
  • Spring Data JPA 활용한 효율적인 데이터베이스 연동
  • Thymeleaf 템플릿 엔진으로 동적 웹 페이지 구현
트러블 슈팅

문제: Spring Boot 의존성 충돌과 DB 연결 불안정 (초기 환경 설정 과정에서 발생)

구체적 상황:

• Spring Boot 3.3.5와 MySQL Connector 버전 불일치로 런타임 에러
• `spring.jpa.hibernate.ddl-auto` 미설정으로 엔티티-테이블 매핑 실패
• HikariCP 미설정으로 다중 접속 시 Connection Pool 고갈

해결: pom.xml 의존성 호환 버전 명시, application.properties에 JPA ddl-auto와 HikariCP maximum-pool-size 설정 추가, 환경별 프로파일 분리

배운 점: 체계적 사전 설계의 중요성 체득. 이후 4학년 캡스톤에서는 1.5개월을 설계에 투자하여 구조 변경 없이 안정적으로 완성

Tech Stack: Java, Spring Boot, Spring Data JPA, MySQL, Thymeleaf, Maven

렌터카 관리 시스템

Java Swing 기반 데스크톱 애플리케이션 | 2024.03 ~ 05
개요
Java GUI 프로그래밍과 데이터베이스 연동 기술을 학습하기 위해 시작한 프로젝트입니다. 렌터카 업체의 회원, 차량, 예약 정보를 효율적으로 관리하고자 MVC 패턴과 DAO 패턴을 적용한 데스크톱 통합 관리 시스템을 개발했습니다.
기여한 부분
  • AbstractBaseDAO 클래스 도입으로 다중 DAO의 DB 연결 관리 문제 해결
  • MVC 패턴 기반 아키텍처로 View-Business Logic 분리
  • Oracle JDBC 연동 및 안정적인 트랜잭션 관리
  • JCalendar 라이브러리 통합으로 직관적인 예약 날짜 선택 UI 구현
  • 회원/차량/예약 통합 관리 기능 개발
트러블 슈팅

문제: 회원DAO, 차량DAO, 예약DAO 등 각 DAO 클래스에서 중복된 DB 연결 코드로 인한 코드 중복과 연결 누수 위험

구체적 상황:

• 각 DAO마다 `driver`, `url`, `user`, `pwd` 상수 중복 선언
• Connection 생성 로직이 10개 이상 메서드에 반복
• 연결 해제 누락 시 Connection Pool 고갈 위험

해결: `AbstractBaseDAO` 추상 클래스 생성 → 모든 공통 연결 정보와 `getConnection()` 메서드를 상속으로 제공. 모든 DAO가 이를 상속받아 연결 로직 재사용

배운 점: 상속을 활용한 코드 재사용과 DRY(Don't Repeat Yourself) 원칙의 실용성 체득

Tech Stack: Java, Java Swing, Oracle DB (JDBC), JCalendar, MVC Pattern, DAO Pattern
