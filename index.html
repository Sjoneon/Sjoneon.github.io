DaySync

AI 기반 스마트 일정 관리 앱 | 캡스톤 프로젝트 | 2025.03 ~ 11
개요
일정 관리의 번거로움을 해결하고자 시작한 프로젝트로, AI 챗봇을 통한 자연스러운 대화로 일정을 관리하고 실시간 교통정보와 날씨를 제공하는 라이프스타일 앱입니다. 사용자가 복잡한 UI 없이 대화만으로 일정을 등록하고 최적 경로를 안내받을 수 있도록 설계했습니다.
Easy - 비즈니스 가치
문제 (Problem)
"내일 오후 2시 강남역 회의" 일정 하나를 관리하려면 캘린더 앱 → 알람 앱 → 지도 앱 → 날씨 앱, 이렇게 4개 앱을 따로따로 실행해야 하는 번거로움
제작 (Solution)
"내일 2시 강남역 회의"라고 한 번만 말하면 일정 등록 + 버스 정보 + 날씨까지 한 번에 보여주는 통합 앱 개발
해결 (Result)
여러 앱을 왔다갔다 하지 않고 하나의 앱에서 대화만으로 일정·교통·날씨 정보를 모두 관리
Deep - 기술적 구현
핵심 기술
• 서버 구축 및 배포: FastAPI로 REST API 서버 구현 후 AWS EC2에 배포
• AI 통합: Gemini 2.5 Flash 모델 Function Calling으로 자연어를 구조화된 일정/알람 데이터로 변환
• 경로 탐색: 네이버 지도 API와 Geocoding으로 출발지-도착지 경로 및 버스 정보 제공
• 날씨 정보: 공공데이터포털 날씨 API 연동 및 기상청 API 폴백 처리
• 안드로이드 앱: Java로 REST API 호출, Gemini 응답 파싱, 실시간 UI 업데이트 구현
트러블 슈팅

문제: 대전 데이터센터 화재로 공공데이터포털 날씨 API 중단

상황: 날씨 정보 조회 기능 전체 마비, 사용자에게 "날씨 정보를 가져올 수 없습니다" 에러 발생

해결: 기상청 API를 대체 엔드포인트로 긴급 추가, 공공데이터포털 장애 시 자동 전환 로직 구현

배운 점: 외부 API 의존 시스템은 반드시 폴백 로직이 필요함을 체득. 단일 장애점(SPOF) 제거의 중요성 학습

Tech Stack: Python (FastAPI), Android (Java), AWS EC2, Gemini AI 1.5 Flash, Naver Maps API, 공공데이터포털, 기상청 API

SOS (Style of Simple)

패션 초보 남성을 위한 온라인 쇼핑몰 | 2024.03~06, 09~11
개요
패션에 관심 없었던 남성들이 옷을 고를 때 겪는 어려움을 해결하고자 시작한 프로젝트입니다. 20~30대 남성을 타겟으로 키워드 기반 추천 시스템을 통해 체형·상황·스타일에 맞는 제품을 쉽게 찾을 수 있도록 개발한 Spring Boot 기반 전자상거래 플랫폼입니다.
Easy - 비즈니스 가치
문제 (Problem)
옷을 사고 싶은데 "캐주얼", "데이트룩"처럼 막연한 단어로만 알고 구체적으로 어떤 옷을 골라야 할지 모르는 패션 초보 남성들의 어려움
제작 (Solution)
미리 등록된 "키 180cm", "데이트", "캐주얼" 같은 키워드를 선택하면 그에 맞는 옷을 추천해주는 온라인 쇼핑몰 개발
해결 (Result)
등록된 키워드를 선택하기만 하면 체형·상황·스타일에 맞는 제품을 자동 필터링하여 쉽게 옷을 고를 수 있도록 지원
Deep - 기술적 구현
계층형 아키텍처
• Controller: HTTP 요청 처리, @RequestParam/@PathVariable로 데이터 수신, Model 객체로 뷰에 데이터 전달
• Service: 비즈니스 로직 구현 (주문 처리, 재고 확인, 회원 가입 등), @Transactional로 트랜잭션 관리
• Repository: JpaRepository 상속으로 기본 CRUD 제공, 커스텀 쿼리 메서드(findByUserId 등) 정의
• Model: @Entity로 테이블 매핑, @OneToMany/@ManyToOne으로 엔티티 관계 설정
핵심 기능 구현
• 회원/판매자 시스템: 세션 기반 로그인, 권한별 접근 제어
• 상품 관리: CRUD 기능, 카테고리별 분류, 키워드 기반 필터링
• 주문 처리: 장바구니 → 주문 → 재고 차감 플로우, Order-OrderDetail 관계 설정
• 키워드 추천: Product-Keyword 다대다 관계로 체형·스타일·상황별 상품 매칭
트러블 슈팅

문제: 초기 환경 설정 문제 - Spring Boot 3.3.5와 MySQL Connector 버전 불일치, HikariCP 미설정

상황: 런타임 에러 반복 발생, Connection Pool 고갈로 다중 사용자 접속 실패

해결: pom.xml에서 호환 버전 명시 (Spring Boot 3.3.5 + mysql-connector-j), application.properties에 HikariCP maximum-pool-size 설정

배운 점: 체계적 사전 설계의 중요성 체득. 이후 캡스톤에서는 1.5개월 설계 투자로 구조 변경 없이 안정적 완성

Tech Stack: Java, Spring Boot 3.3.5, Spring Data JPA, MySQL, Thymeleaf, Maven, HikariCP

렌터카 관리 시스템

Java Swing 기반 데스크톱 애플리케이션 | 2024.03 ~ 05
개요
Java GUI 프로그래밍과 데이터베이스 연동 기술을 학습하기 위해 시작한 프로젝트입니다. 렌터카 업체의 회원, 차량, 예약 정보를 효율적으로 관리하고자 MVC 패턴과 DAO 패턴을 적용한 데스크톱 통합 관리 시스템을 개발했습니다.
Easy - 비즈니스 가치
문제 (Problem)
렌터카 업체에서 여러 회원의 예약, 차량 관리를 종이나 엑셀로 하다 보니 실수가 많고 검색이 불편함
제작 (Solution)
회원 정보, 차량 정보, 예약 현황을 한 프로그램에서 등록·조회·수정·삭제할 수 있는 데스크톱 관리 프로그램 개발
해결 (Result)
클릭 몇 번으로 예약 현황을 즉시 확인하고 달력에서 날짜를 선택하여 직관적으로 예약 가능
Deep - 기술적 구현
핵심 기술
• 디자인 패턴: MVC 패턴으로 View-Controller-Model 분리, DAO 패턴으로 데이터 액세스 추상화
• 코드 재사용: AbstractBaseDAO 추상 클래스로 DB 연결 로직 상속, 모든 DAO가 getConnection() 재사용
• 데이터베이스: Oracle JDBC로 연결, PreparedStatement로 SQL Injection 방지
• UI/UX: JCalendar 라이브러리로 달력 기능, JTable로 데이터 시각화
트러블 슈팅

문제: 회원DAO, 차량DAO, 예약DAO에서 DB 연결 코드 중복 (driver, url, user, pwd 반복 선언)

상황: Connection 생성 로직이 10개 이상 메서드에 반복, 연결 해제 누락 시 Connection Pool 고갈 위험

해결: AbstractBaseDAO 추상 클래스에 공통 연결 정보와 getConnection() 정의, 모든 DAO가 상속받아 코드 재사용

배운 점: 상속을 활용한 코드 재사용과 DRY 원칙의 실용성 체득

Tech Stack: Java, Java Swing, Oracle DB (JDBC), JCalendar, MVC Pattern, DAO Pattern

최적 경로 탐색 시스템

Dijkstra 알고리즘 기반 경로 탐색 | 4인 팀 프로젝트 | 2023.03 ~ 06
개요
교통수단 선택의 어려움을 해결하기 위해 Dijkstra 알고리즘을 활용한 최적 경로 계산 프로그램입니다. 자동차, 버스, 도보 등 다양한 교통수단의 비용과 시간을 고려하여 최적 경로를 제시합니다.
Easy - 비즈니스 가치
문제 (Problem)
여러 교통수단 중 어떤 것이 가장 효율적인지 판단하기 어려움
제작 (Solution)
출발지와 목적지를 입력하면 교통수단별 최적 경로와 비용을 자동 계산
해결 (Result)
시간과 비용을 고려한 최적의 이동 경로 제공
Deep - 기술적 구현
핵심 기술
• 알고리즘: Dijkstra 알고리즘 구현, 시간 복잡도 O((V+E)logV)
• 자료구조: 그래프를 인접 리스트로 구현, 우선순위 큐 활용
• 팀 역할: TUI 인터페이스 단독 개발, 알고리즘 로직 30% 기여
• 협업: Git을 통한 버전 관리, 팀원과 코드 리뷰 및 통합
학습 성과

자료구조와 알고리즘을 단순히 이론으로만 아는 것이 아니라 실제로 구현하며 깊이 이해할 수 있었습니다. 특히 그래프를 인접 리스트로 구현하여 메모리 효율성을 높였고, 우선순위 큐를 활용해 알고리즘의 시간 복잡도를 최적화했습니다. 4명이 함께 작업하며 Git을 통한 버전 관리와 코드 통합 경험도 쌓았습니다.

Tech Stack: C, Dijkstra 알고리즘, 그래프 자료구조, 우선순위 큐, Git
